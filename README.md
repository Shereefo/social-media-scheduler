## Overview
This project is a social media scheduler that enables CRUD (Create, Read, Update, Delete) operations for scheduling posts. It uses **FastAPI** to build a RESTful API with automatic interactive documentation provided by **Swagger UI**. The system is now fully containerized using **Docker** and orchestrated with **Docker Compose**, featuring a **PostgreSQL** database for persistent storage. 

### **Latest Enhancements**:
- ✅ **Database Integration**: Migrated from SQLite to PostgreSQL for better scalability.
- ✅ **Dockerization**: Fully containerized with a `Dockerfile` and `docker-compose.yml`.
- ✅ **GitHub Repository**: Version control and cloud backup enabled.
- ✅ **Automated Testing**: API tested via **Swagger UI** and **cURL commands**.
- ✅ **Future Expansion**: Planning integration with **TikTok API** for real-world automation.

## Setup & Installation

1. **Clone the Repository:**
   ```bash
   git clone https://github.com/Sherefo/social-media-scheduler.git
   cd social-media-scheduler
   ```

2. **Create & Activate Virtual Environment:**
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Run the Application Locally:**
   ```bash
   uvicorn backend.app.main:app --reload
   ```

5. **Access API Documentation:**
   - Open [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs) to interact with the Swagger UI.

## API Endpoints

### 📋 **Root Endpoint**
- **GET /**  
  Returns a welcome message to verify the server is running.

### 📅 **Post Management Endpoints**

1. **Create Post**  
   - **POST /posts/**  
   - Adds a new scheduled post to the database.

2. **Get All Posts**  
   - **GET /posts/**  
   - Retrieves all scheduled posts.

3. **Get Post by ID**  
   - **GET /posts/{post_id}**  
   - Retrieves a specific post by ID.

4. **Update Post**  
   - **PUT /posts/{post_id}**  
   - Updates an existing scheduled post.

5. **Delete Post**  
   - **DELETE /posts/{post_id}**  
   - Deletes a scheduled post by ID.

## Docker Setup

### Running the Application in Docker

To start both the FastAPI app and PostgreSQL database:

```bash
docker-compose up --build -d
```

To stop the containers:

```bash
docker-compose down
```
